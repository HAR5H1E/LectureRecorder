# CSC-335 Team Project Overview

## Project Deadline
The team project must be completed by **Monday, 4 May 2026**, at 11:59 PM.

## Project Description for the Rest of the Turn
To accomplish one of the four tasks below. Teams will be made up of four students and will work using an Agile methodology.

### Tasks

**Task 1:** Monopoly - Design and implement a digital version of Monopoly, where players roll dice, purchase properties, and compete to bankrupt opponents. The game features a graphical board, an automated banking system, and customizable rules to enhance gameplay.

```plaintext
- **Graphical Board**
- **Automated Banking System**
- **Customizable Rules**
```

**Task 2:** Redsheets - Application that allows users to input data, apply formulas, and format cells in a grid-based interface. File saving, loading, and CSV export will be supported.

```plaintext
- **File Saving**
- **File Loading**
- **CSV Export**
```

**Task 3:** Interactive Planner - Create a customizable digital planner for managing tasks, organizing schedules, and personalizing layouts.

## Project Requirements

### Key Components of Your Final Grade
Your project final grade will be based on:
- **60%** on the actual software artifact you produce.
- **40%** on your participation in and success with the Agile methodology we outlined in class and below.

### Sprint Breakdown
Weekly stand-up meetings will follow a Scrum style. They will minimally cover:

```plaintext
- Improvements to print quality
- Potential blocking items
```

### Meeting Notes
You should take detailed notes describing how your weekly progress went. These notes should be used both to document the team's progress and keep the team cognizant of any discussion.

## Project Documentation

Each sprint planning, sprint review, and sprint retrospective meeting will serve as a brief outline of the meeting’s content and should be documented accordingly. Here is the breakdown:

- **50%** Wild Factor
- **20%** Team Process Documentation
  - Sprint Planning
  - Sprint Review
  - Sprint Retrospective
- **10%** GitHub Integration

## Project Evaluation

### Self-Team Assessment Presentation
There will be a presentation on the self-assessment and team assessment of your project. Failure to participate in any week’s meeting will result in a zero for the final grade.
